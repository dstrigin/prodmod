# Продукционная модель представления знаний

--------------------------------------------

Тема: выявления причин поломки авто и советы по их устранению

Все необходимые данные расположены в директории `data/`

## Запуск

```shell
go run ./cmd/
```

## Данные для работы

Необходимо несколько файлов: факты, правила, аксиомы, целевые факты

1. `facts.txt` (База фактов)

Справочник всех возможных фактов, которые известны системе (симптомы, промежуточные выводы, рекомендации).
   
Формат: ID;Описание;Вес

Пример:
```text
f01;Двигатель не запускается;0.9
f30;Проблема с аккумулятором;0.8
f100;Зарядить аккумулятор;0.9
```

2. `rules.txt` (База правил)
Логика, связывающая факты. Описывает переход вида: ЕСЛИ (А и Б) ТО (В).
   
Формат: ID;ВходящиеФакты;Результат;Описание;Вес
   
Пример:
```text
r01;f01,f02;f30;Двигатель не запускается + Стартер не крутит -> Проблема с аккумулятором;0.9
r63;f40,f30;f101;Разряжен аккумулятор + Проблема с аккумулятором -> Заменить аккумулятор;0.85
```

4. `axioms.txt` (Входные данные)
   
Список фактов, которые уже известны (наблюдаемые симптомы). Это стартовая точка для алгоритма.

Формат: Список ID фактов (по одному на строку).
   
Пример:
```text
f01
f02
f04
```

5. `target.txt` (Цели поиска)

Список фактов-рекомендаций, которые система пытается доказать или опровергнуть.

Формат: Список ID фактов (по одному на строку).

Пример:
```text
f100
f101
f102
```
